# 6A Workflow

### Activation

When the user input starts with `@6A`, output exactly:

> ✅ 6A 前端开发流程已激活
> 当前任务类型：【新增功能开发】
> 当前阶段：Align（目标对齐）

### Stage ownership

- Align: `intake` + `planner`
- Architect: `planner`
- Atomize: `planner`
- Approve: `review-gate`
- Automate: `orchestrator` dispatching `engineering`, `docs-spec`, `platform`, and `security` as needed
- Assess: `orchestrator` + `docs-spec`

### Required documents

- `docs/[YYYY_MM_DD]_[中文任务名]_vN/01_ALIGNMENT_[中文任务名].md`
- `docs/[YYYY_MM_DD]_[中文任务名]_vN/02_CONSENSUS_[中文任务名].md`
- `docs/[YYYY_MM_DD]_[中文任务名]_vN/03_DESIGN_[中文任务名].md`
- `docs/[YYYY_MM_DD]_[中文任务名]_vN/04_TASK_[中文任务名].md`
- `docs/[YYYY_MM_DD]_[中文任务名]_vN/05_APPROVE_[中文任务名].md`
- `docs/[YYYY_MM_DD]_[中文任务名]_vN/06_ACCEPTANCE_[中文任务名].md`
- `docs/[YYYY_MM_DD]_[中文任务名]_vN/07_FINAL_[中文任务名].md`
- `docs/[YYYY_MM_DD]_[中文任务名]_vN/08_TODO_[中文任务名].md`

### Document bootstrap

- create or update the required files under the active project's root `docs/`
- default the bundle version to `v1`, then increment to `v2`, `v3`, and so on for later revisions
- `intake` must place the required-document list into the first task card
- `planner` must turn the list into a staged document plan
- `orchestrator` must dispatch `docs-spec` when files are missing or outdated
- block implementation until the document skeleton set exists
- for code-changing work, document file paths, line ranges, before context, and after context

### Control rules

- do not skip stages
- do not start coding before Align and Architect are complete
- if approval fails, return to Architect or Atomize
- each atomic task should map to one executable change unit where feasible
